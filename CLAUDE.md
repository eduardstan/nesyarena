## What this project is

NeSyArena measures **semantic fidelity**: the signed, oracle-grounded error between what a neuro-symbolic (NeSy) reasoning approximation computes and what its *claimed semantics* defines, as a function of program structure — plus the learning consequences (gradient liveness, calibration against exact Bayes posteriors, held-out transfer). It grew out of a KR2026 theory submission (three weak rejects) that is being pivoted into a measurement instrument + empirical paper.

Authoritative documents, in order:
- `NeSyArena_protocol_v1.md` — formal definitions (D1–D6), hypothesis table (H1–H10), generators (G1–G4), experiment matrix (E1–E8), review-risk mapping
- `out/RESULTS.md` — all measured results from the toy implementation
- `paper/main.tex` — comprehensive technical report (definitions, laws+proofs, methodology, all measured results, findings, limitations, FAQ); compiles to PDF

## Architecture (rebuild, `src/nesyarena/`)

Data flow: **generator → program (IR) → enumerated proofs → both the oracle and each SUT, fed the identical base interpretation → signed error**. That shared-input invariant is what isolates the reasoning layer from perception.

- `ir.py` — `Atom`/`Rule`/`GroundProgram` + bounded proof enumeration (leaf-multiset view for semiring evaluation, support-set view as the DNF for WMC/SUTs). Programs are the primary object so external adapters compile the same instance the oracle scores.
- `oracle.py` — ground truth. Brute-force WMC (≤ 22 facts) with *analytic* gradients (`dP/dp_f = P(q|f) − P(q|¬f)`); ProbLog exact inference as the >22-fact oracle; direct graph oracles (boolean/tropical/max-product).
- `suts.py` — reference SUTs mirroring deployed NeSy aggregations (add-mult clamped/raw, top-k, min-max, LSE), each with **system-faithful gradients** (top-k freezes selection, min-max routes a one-hot subgradient — deterministic lexicographic tie-break, a documented deviation from the toy whose tie depended on PYTHONHASHSEED; clamp has a flat region). `claimed` declares the oracle: WMC for probabilistic SUTs, max-join for LSE.
- `algebra.py` / `engine.py` — semiring registry; T_P iteration (Defs. 11–13), run-to-convergence (tracks the whole interpretation, not just the query), proof-side aggregation with the Theorem-1 correspondence as a standing numeric test.
- `generators.py` — G1 overlap (P proofs × length L × c trunk facts), G2 chain/cyclic transitive closure, G3 surrogate score multisets. Fact naming and rng draw order replicate the toy exactly.
- `metrics.py` / `witness.py` — fidelity profile, depth horizon, gradient liveness; minimal-witness grid+shrink.

Parity discipline: `tests/fixtures/toy_golden.json` (generated by `legacy/gen_golden_from_toy.py`) pins toy behavior; parity tests replay it. Toy and rebuild share the `nesyarena` import name — never put both on one `sys.path`; fixtures exist precisely so cross-checks happen via files, not co-imports.

## Project-specific rules

- **Discrepancies between an external system (Scallop, ProbLog, DeepProbLog, DeepLog) and the reference SUTs are findings about that system's deployed semantics — log them with the instance; never "fix" them away or normalize them.**
- The hypothesis-table discipline is part of the artifact's credibility: predictions (sign, growth law) are registered *before* runs; measured outcomes update the H-table either way. A refuted hypothesis (e.g. H-A in `run_ablation.py`) is a reportable result.
- Oracle changes must keep the CI battery green to ~1e-10 agreement (reference WMC ≡ ProbLog); the oracle never silently rots.
- Terminology (binding, from the KR review response): say "claimed semantics", "semantic error", "conformance", "witness". Do not use "contract", "audit", "certificate" in paper-facing text.
